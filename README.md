# Database-Session-Task-Semester-2

## Краткое описание проекта

Этот проект представляет собой набор из 4 задач с разными кол-во подзадач. Всего используется 4 базы данных каждая для своей задачи.  В ходе решения задачь работала со стеком phpmyadmin. 
Каждая директория проекта посвещана своей задачи. Внутри каждой директории есть типовые папки общие для всех:

- SQL-скрипты для создания таблиц - Scripts_to_create_tables
- SQL-скрипты для заполнения таблиц тестовыми данными - Scripts_for_filling_data
- набор конкретных подзадач с готовыми решениями - task_to_check



## Структура проекта

Проект состоит из четырех самостоятельных блоков:

- `Ex_1_vehicles` — база данных по транспортным средствам;
- `Ex_2_autorace` — база данных по автомобильным гонкам;
- `Ex_3_booking` — база данных по бронированию отелей;
- `Ex_4_orgstructure` — база данных по структрной организаии.

Внутри каждого блока используются две основные папки:

- `Data_preparation` — скрипты создания таблиц и заполнения тестовыми данными;
- `Task_to_check` — SQL-задачи и их решения.

## Структура баз данных

### 1. `Ex_1_vehicles`

Основные таблицы:

- `Vehicle` — общая информация о модели и производителе;
- `Car` — характеристики автомобилей;
- `Motorcycle` — характеристики мотоциклов;
- `Bicycle` — характеристики велосипедов.

Что реализовано:

- работа с несколькими типами транспорта;
- фильтрация по характеристикам;
- объединение данных из нескольких таблиц;
- сортировка и `UNION ALL`.

### 2. `Ex_2_autorace`

Основные таблицы:

- `Classes` — классы автомобилей;
- `Cars` — автомобили;
- `Races` — гонки;
- `Results` — результаты участия автомобилей в гонках.

Что реализовано:

- расчёт средней позиции автомобилей;
- ранжирование внутри классов;
- поиск лучших и худших классов;
- использование `WITH`, `RANK()` и агрегатных функций.

### 3. `Ex_3_booking`

Основные таблицы:

- `Hotel` — отели;
- `Room` — номера;
- `Customer` — клиенты;
- `Booking` — бронирования.

Что реализовано:

- анализ активности клиентов;
- подсчёт количества бронирований и суммы расходов;
- определение предпочтений по типам отелей;
- использование `GROUP_CONCAT`, `AVG`, `SUM`, `DATEDIFF`.

### 4. `Ex_4_orgstructure`

Основные таблицы:

- `Departments` — отделы;
- `Roles` — роли сотрудников;
- `Employees` — сотрудники и их иерархия;
- `Projects` — проекты;
- `Tasks` — задачи сотрудников.

Что реализовано:

- работа с иерархическими данными;
- поиск подчинённых по рекурсивной структуре;
- подсчёт количества задач и подчинённых;
- использование `WITH RECURSIVE`.

## Основные функции и реализованные возможности

В проекте реализованы следующие типы SQL-решений:

- создание схем баз данных;
- заполнение таблиц тестовыми данными;
- выборка и фильтрация записей;
- объединение данных из нескольких таблиц;
- группировка и агрегирование;
- рекурсивные запросы;
- сортировка и ранжирование результатов.

## Инструкция по запуску проекта

Проект не требует отдельного серверного приложения. Для запуска достаточно любой СУБД, совместимой с используемым SQL-синтаксисом. Лучше всего подходит `MySQL 8+`, так как в проекте используются:

- `WITH` и `WITH RECURSIVE`;
- оконные функции, например `RANK()`;
- `GROUP_CONCAT`;
- `ENUM`.

### Порядок запуска

Для каждого задания рекомендуется выполнять шаги в следующем порядке:

1. Создать пустую базу данных в MySQL.
2. Открыть файл `Scripts_to_create_tables.md` и выполнить SQL-команды из блоков `sql`.
3. Открыть файл `Scripts_for_filling_data.md` и выполнить SQL-команды для заполнения таблиц.
4. Открыть нужный файл из папки `Task_to_check`.
5. Выполнить SQL-запрос из раздела `Решение`.

## Инструкция по тестированию

Для проверки корректности работы:

1. Выполните создание таблиц.
2. Загрузите тестовые данные.
3. Запустите SQL-запрос из нужной задачи.
4. Сравните результат с логикой, описанной в разделе `Ожидаемы результаты`.
